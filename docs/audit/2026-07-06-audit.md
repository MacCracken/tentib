# tentib 0.8.0 ‚Äî security / hardening audit

**Date:** 2026-07-06 ¬∑ **Scope:** the frozen public surface (`docs/api.md`) at
0.7.0, suite 97/97 going in ¬∑ **Method:** re-derive from source ‚Äî every claim
below was verified by reading the implementation, not its comments (the
audit-re-derive discipline). Cyrius has no bounds checks; the substrate contract
(rosnet) is caller-guarantees, so the finding bar is *silent heap corruption or
silently-wrong numerics reachable through a public entry point*.

## Posture decision

tentib follows the **tarka precedent** (fail-loud `guard()` on public-API
preconditions, cold paths only), not the rosnet one (audited-deliberate bare
contract): tentib is a reference *binary* whose public surface is meant to be
called by consumers (hoosh/murti serving), and the hazards found were of the
silently-*wrong* kind, which no caller can detect downstream. Hot inner loops
carry **zero** added checks ‚Äî bit-identity and the ¬ß1 sweep numbers are
unchanged; the packed serving path pays one compare per *call*.

## Out of scope / accepted contract

- **No untrusted-input surface**: no file parsing, no network, no deserialization
  (see `SECURITY.md` threat model) ‚Äî the anuk≈´lana-style hostile-input audit
  dimension does not exist here.
- **Buffer sizing stays caller-guarantees** (`tunpack2` index, y/qx/scratch
  extents) ‚Äî documented per symbol in `api.md`; guarding element accessors would
  put checks in hot loops for hazards a consumer controls statically.
- `rng_u64() % n` negative-draw gotcha: tentib's only modulo-of-rng is in the
  bench harness, already masked (`& 0x7FFF‚Ä¶`).
